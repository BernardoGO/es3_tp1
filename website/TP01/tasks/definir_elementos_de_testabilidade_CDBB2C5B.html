Task: Definir Elementos de Testabilidade
Purpose

Alguns elementos não necessitam de teste, ou não obteriam ganho real de desempenho, portanto definir elementos chave diminui tempo de desenvolvimento

Relationships
RolesPrimary Performer:
  • analista de teste
Additional Performers:
InputsMandatory:
  • Arquitetura do Teste de Automacao
    		Optional:
    • None
    Process Usage
    • teste  >  Testes  >  Testar_e_Avaliar  >  Definir Elementos de Testabilidade
    Steps
    Para cada mecanismo de teste necessário, identificar as relações com os itens de destino
    Obter uma compreensão de como cada mecanismo de teste necessário terá de suportar os itens de teste de destino
    Identificar limites do sistema e interfaces
    Obter uma compreensão das responsabilidades do sistema como um provedor de serviços, e as dependências do sistema como um cliente
    Identificar elementos da infra-estrutura de teste
    Identificar os elementos essenciais do esforço de teste que permitirão realizar os ensaios necessários
    Definir requisitos de testabilidade de software
    Especificar os requisitos para as funções de software necessárias para suportar a implementação e execução de testes
    Avaliar e verificar seus resultados
    Para verificar se a atividade foi concluída adequadamente e se os artefatos resultantes são aceitáveis
